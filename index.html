网页版说明

这一版优先把项目“产品化”为网页，因此把原仓库中的诊断树规则前移到浏览器端执行。 现在也已经支持接入 Python 后端和模型 API，可以直接在网页上触发真实诊断流程。

如果后续要接大模型接口，我们可以再把现有 `methods.py` 封装成 API 服务。

演示建议

答辩时可先用知识图谱展示规则路径，再切换到后端模型分析，说明系统同时具备规则可解释性与模型诊断能力。

这会比单纯展示一个诊断结果，更像完整的临床决策支持系统。
