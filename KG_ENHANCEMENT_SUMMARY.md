# 知识图谱增强功能实现总结

## 概述

已实现从 `/data/cardiovascular_files` 目录下的住院病例文件中提取临床术语变体，通过网页界面交由医生进行审核，最终将审核通过的术语合并到知识图谱中的完整功能。

## 实现内容

### 1. 后端模块 (`backend/kg_enhancement.py`)

#### 核心类

1. **ExtractedTerm** - 数据类
   - 表示从病例中提取的临床术语
   - 包含术语、类别、来源文件、频率、审核状态等属性

2. **TermExtractor** - 术语提取器
   - 使用正则表达式从文本中提取临床术语
   - 支持 4 类术语：症状、临床发现、检查、诊断
   - 内置丰富的医学术语模式

3. **SemanticMatcher** - 语义匹配器
   - 将提取的术语与知识图谱实体进行匹配
   - 使用字符串相似度算法计算匹配置信度
   - 支持预定义同义词组匹配

4. **KGEnhancementManager** - 知识图谱增强管理器
   - 管理完整的术语提取、审核、合并工作流
   - 支持状态持久化和导出增强知识图谱

#### 术语提取模式

已内置以下术语模式（部分示例）：

**症状 (symptom)**:
- 胸痛、胸闷、心悸、气促
- 大汗淋漓、恶心呕吐
- 呼吸困难、晕厥、紫绀

**临床发现 (finding)**:
- ST 段抬高/压低/改变
- T 波倒置/高尖/改变
- 肌钙蛋白升高/阳性
- CK-MB 升高、BNP 升高

**检查 (exam)**:
- 心电图、ECG、Holter
- 超声心动图、心脏彩超
- 冠脉 CTA、冠状动脉造影

**诊断 (diagnosis)**:
- STEMI、NSTEMI、不稳定型心绞痛
- 变异型心绞痛、急性冠脉综合征
- 肺栓塞、心力衰竭、心律失常


### 2. API 接口 (`backend/server.py`)

已添加 7 个 API 端点：

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/api/kg-enhancement/status` | 获取服务状态 |
| POST | `/api/kg-enhancement/extract` | 从病例提取术语 |
| GET | `/api/kg-enhancement/review-items` | 获取审核项目列表 |
| POST | `/api/kg-enhancement/review` | 提交医生审核 |
| POST | `/api/kg-enhancement/merge` | 合并术语到知识图谱 |
| POST | `/api/kg-enhancement/export` | 导出增强知识图谱 |
| GET | `/api/kg-enhancement/download/<filename>` | 下载知识图谱文件 |

### 3. 前端界面 (`frontend/kg-enhancement.html`)

#### 主要功能

1. **状态面板**
   - 显示提取术语数、待审核组数、已审核通过数、图谱实体数

2. **筛选功能**
   - 按类别筛选：症状、临床发现、检查、诊断
   - 按状态筛选：待审核、已通过、已拒绝
   - 按匹配置信度筛选：高、中、低

3. **术语审核**
   - 显示术语变体列表
   - 显示出现上下文
   - 支持单个审核（通过/拒绝）
   - 支持批量通过
   - 记录审核医生姓名和意见

4. **知识图谱合并**
   - 一键合并所有已审核通过的术语
   - 显示合并统计信息

#### 界面特点

- 响应式设计，适配不同屏幕尺寸
- 温暖的医学主题配色
- 实时状态反馈
- 本地存储审核医生姓名

### 4. 导航集成

在主页面 (`frontend/index.html`) 添加了"知识图谱增强"入口链接。

## 使用流程

### 1. 启动服务
```bash
python server.py
```

### 2. 访问界面
打开浏览器访问：`http://localhost:8000/kg-enhancement.html`

### 3. 提取术语
点击"提取术语"按钮，系统自动扫描病例文件

### 4. 医生审核
- 查看提取的术语变体
- 点击"通过"或"拒绝"
- 可填写审核意见

### 5. 合并到图谱
审核完成后，点击"合并到图谱"
